실제 구현 시 동작 방식

이 데모가 시뮬레이션하고 있는 실제 흐름입니다.

1. 업로드
브라우저가 POST /api/v0/add 로 파일을 자체 IPFS 노드(sailab06)에 전송합니다. 노드는 내용 해시 기반의 CID를 반환하고, pin=true 로 영구 보관합니다.

2. 발행
받은 CID와 메타데이터를 인자로 mint(cid, title, description, mimeType, fileSize) 를 호출합니다. ESM 체인(chainId 7604)이 2초 안에 블록을 확정하고 Minted 이벤트를 남깁니다.

3. 열람
NFT의 tokenId로 컨트랙트에서 CID를 조회한 뒤, IPFS 게이트웨이에서 원본을 렌더링합니다. 파일 자체는 체인에 올라가지 않으므로 용량 제약이 없습니다.

4. 조회 기록
logView(tokenId) 호출 시 ViewLogged 이벤트가 발생합니다. 누가 언제 무엇을 열람했는지가 체인에 남아 위변조 없는 감사 로그가 됩니다. 대시보드는 이 이벤트를 집계해 표시합니다.
